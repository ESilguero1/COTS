① Command Path — Python Host → Motor
🖥
Python Host
Builds CmdMessenger string e.g. 30,0,45000;
Sends over USB Serial (COM3 · 115200 baud)
CMD
↓USB CDC Serial · ASCII frame ending with ;
📡
CmdMessenger · feedinSerialData()
Called every loop() iteration · parses command ID and args · dispatches to registered callback
PARSE
↓callback dispatch by command enum (10–49)
⚙️
System_Control_App — Callback
e.g. onConstantForward() · validates flags · reads motor ID + velocity args · checks seek/position state
APP
↓direct method call
🔧
CombinedControl
Range check · ramp mode selection · delegates to MotorControl[n]
CTRL
↓MotorControl method call
📦
MotorControl[n] — sendData()
Builds 40-bit SPI datagram (WRITE bit | address, 32-bit data)
Asserts CS pin · transfers 5 bytes · de-asserts CS
SPI
↓SPI MODE3 · MSBFIRST · CLK/8 · individual CS pins
⚡
TMC5130 Register Write
e.g. VMAX (0x27) · RAMPMODE (0x20) · XTARGET (0x2D)
Motor executes motion profile autonomously
HW
↑acknowledgement bubbles back up
✅
onSuccess() / onFail()
Writes response to Serial + BLE simultaneously
S,1; (success) S,0; (fail) e, unknown command;
ACK
Command enum ranges: 10–12 = requests · 20–27 = getters · 30–39 = setters · 40–49 = control ops
Every response ends with ; and is mirrored to both Serial and BLE_App_sys.println()
② IMU Telemetry Path — Sensor → Python
🧭
HWT906 IMU
6-axis AHRS · outputs accel, gyro, mag frames
Connected via UART1 (pins 18/19)
HW
↓UART1 RX interrupt · binary frame protocol
📥
HWT906_App · CheckIMUDataCollection()
Called by AsyncTask every 233 ms
Parses frames · accumulates samples into AveragedIMUdata[6]
Counts communication errors → IMU_Comm_Errors
COLLECT
↓extern AveragedIMUdata[6] (floatUnion) · extern IMU_Comm_Errors
📊
onGetIMUData() — triggered by GET_IMU_AVE_DATA (cmd 27)
Reads motor standstill + position reached for all 3 motors
Packs into positionReached byte (bits 0,1,4,5) + PositionReachedM3 byte (bits 0,1)
PACK
↓sprintf hex encoding per axis (floatUnion.i → hex string)
📤
Serial.println() + BLE_App_sys.println()
Dual-channel output: USB CDC + BLE UART
imu,hexAx,hexAy,hexAz,hexGx,hexGy,hexGz,errCnt,M3stat,M12stat;
TX
↓USB Serial or BLE UART
🖥
Python Host
Parses imu frame · decodes hex floats · uses motor status bits for closed-loop control
RECV
positionReached byte layout:
bit 0 = M0 standstill (vzero) · bit 1 = M0 posReached
bit 4 = M1 standstill (vzero) · bit 5 = M1 posReached
PositionReachedM3: bit 0 = M2 standstill · bit 1 = M2 posReached
③ SPI Status Readback — TMC5130 Register Pipeline
🔍
MotorControl::readStatus()
Called via CombinedControl::status() · reads RAMP_STAT + DRV_STATUS + GCONF + GSTAT
READ
TMC5130 SPI pipeline — each response = result of previous read request:
T1: RAMP_STAT
← stale
T2: RAMP_STAT
← RAMP_STAT✓
T3: RAMP_STAT
← RAMP_STAT✓
T4: DRV_STATUS
← RAMP_STAT ★
T5: GCONF
← DRV_STATUS ★
T6: GSTAT
← GCONF✓
T7: GSTAT
← GSTAT✓
★ = where bit extraction happens · T1 is pipeline flush
📋
RAMP_STAT (0x35) — extracted at T4 response
bit 0: stop_l · bit 1: stop_r · bit 2: latch_l · bit 3: latch_r
bit 6: sg_event · bit 7: pos_reached_event · bit 8: vel_reached
bit 9: position_reached · bit 10: vzero → status_standstill · bit 13: sg
RAMP
📋
DRV_STATUS (0x6F) — extracted at T5 response
bit 25: overtemp_shutdown · bit 26: overtemp_warning
bit 27: shortToGround_B · bit 28: shortToGround_A
bit 29: openLoad_A · bit 30: openLoad_B · bit 31: stst (not used for standstill)
DRV
Key fix: standstill now reads RAMP_STAT bit 10 (vzero — VACTUAL=0, immediate)
rather than DRV_STATUS bit 31 (stst — timer-based, ~87 ms lag at 12 MHz clock).
getVelocity() reads VACTUAL (0x22) live from chip, not the cached VMAX write-register.
